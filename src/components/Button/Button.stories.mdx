# Button

With `MDX` we can define a story for `Button` right in the middle of our
markdown documentation.

<Story name="all buttons">
  <TransparentWrapper transparentGB={true}>
    <form>
      <Button>Participer</Button>
      <Button variant="outlined">Participer</Button>
      <Button color="secondary">Participer</Button>
      <Button loading={true}>Loading</Button>
      <Button
        loading={true}
        variant="outlined"
      >
        Loading
      </Button>
    </form>
  </TransparentWrapper>
</Story>


We can even preview multiple stories in a block. This
gets rendered as a group, but defines individual stories
with unique URLs and isolated snapshot tests.

<Preview>
  <Story name="without props">
      <Button>Participer</Button>

        Loading
      </Button>
  </Story>
</Preview>
